# MedNexus on Cloudflare (Workers + D1 + Assets)

A rewrite of the original `professional-api-server` (Node/Express + better-sqlite3 + argon2)
for **Cloudflare Workers**, using:

- **Hono** as the HTTP framework (edge-native, pure JS)
- **Drizzle ORM** with the **D1** (SQLite) driver
- **Workers Assets** to serve the built SPA from the same origin as the API
- **Web Crypto (PBKDF2)** for password hashing (argon2 is unavailable on Workers)
- Same-origin cookie sessions (no JWT required)

## Project layout

```
src/
  worker.ts            # Hono app: middleware, routes, SPA fallback
  types.ts             # Bindings / Env types
  db/schema.ts         # Full Drizzle schema (ported from the original)
  db/index.ts          # D1 drizzle instance + table exports
  lib/auth.ts          # sessions, password hashing, Google token verify
  lib/config.ts        # env-based config
  lib/password.ts      # PBKDF2 hashing (Web Crypto)
  middleware/validate.ts # Zod validation + shared schemas
  routes/              # auth, decks, cards, health
migrations/            # D1 SQL migration (generated by drizzle-kit)
public/                # prebuilt frontend (copied from new-frontend/dist)
wrangler.toml          # Worker / D1 / Assets config
```

## What works (ported from the original)

- `GET  /healthz`, `GET /api/health`, `GET /api/model-info`
- Auth: `guest`, `user`, `login`, `register`, `logout`, `logout-all`, `oauth/google`
- Decks: list, create, tree, get, patch, delete, cards, export, merge, move
- QBank tree (`/api/qbanks/tree`)
- Cards: list (by `deckId`), create, patch, delete, `regenerate-batch`
- The **complete 40-table schema** is created in D1.

## Not yet implemented (schema present, routes pending)

These original features need deeper rework for the edge (no filesystem, no native
modules, no SMTP). The tables exist so data is preserved; endpoints return 404 until added:

- Terminal / agent workspaces (needs R2 or D1-backed "files")
- PDF / DOCX / OCR ingestion (tesseract.js, pdfjs, mammoth need Node builtins)
- Email sending (needs an HTTP email provider e.g. Resend/Mailchannels instead of nodemailer)
- Microsoft / Apple OAuth (need Web Crypto RSA verify)
- Study plans, exams, notifications, tags, achievements, settings, support, chat, article jobs

## Deploy steps (require `wrangler` authentication)

```bash
cd /home/khaled/Desktop/cloudflare

# 1. Authenticate the CLI (opens a browser). Alternatively set
#    CLOUDFLARE_API_TOKEN in your environment.
npx wrangler login

# 2. Create the D1 database and copy the printed id into wrangler.toml
#    (replace REPLACE_WITH_YOUR_D1_ID).
npx wrangler d1 create mednexus-db

# 3. Apply the schema migration (remote). Add --local to test locally first.
npx wrangler d1 migrations apply mednexus-db

# 4. (Optional) Set secrets used by AI features / admin / Google login:
npx wrangler secret put OPENROUTER_API_KEY
npx wrangler secret put OPENAI_API_KEY
npx wrangler secret put GROQ_API_KEY
npx wrangler secret put GOOGLE_CLIENT_ID
npx wrangler secret put ADMIN_SECRET_KEY

# 5. Copy the frontend build and deploy.
npm run deploy          # = build:frontend + wrangler deploy
```

## Local development

```bash
npm run dev             # build:frontend + wrangler dev (local D1)
```

The frontend is served from the same origin as `/api`, so no CORS changes are needed.
